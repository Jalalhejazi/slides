I work on PhoneGap.

  • You probably know, with PhoneGap anyone can build apps with html, css, and javascript.
  • Also a powerful extensibility w/ plugins: access ANY native API from a browser..

Did the web solve cross platform?

  • Most devices, screens, and operating systems.
  • Tell the iPhone 1.x apps would be web apps story.
  • I loved this so much, a vision where we could built to a platform but not be locked into it.
  • We could resuse our skills, infra, and code.
  • We asked ourselves, why not the web?

The web did not solve everything.

  • We're going to ignore browser interop. Thats a problem but its quickly going away.
  • The offline story is a bit of a mess: cache manifest, websql to indexeddb, web storage, no fs.
  • Sandboxed from the host Operating Sytem: consequence of this feature: no device apis.
  • Cannot particpate in App Store revenue or discovery. untapped story. crucial channel for adoption.

Web Operating Systems

  • Did you notice? You have a front row seat for the dawn of web operating systems.
  • Microsoft Windows 8
  • Google Chrome OS
  • Mozilla Firefox OS
  • Tizen (Samsung and Intel sponsored)
  • HP Open webOS

PhoneGap can target all Operating Systems, and specifically mobile first.

  • Interestingly, most app stores are all hybrid apps themselves.

Pls forgive me if you've heard this one before!

  • → Macromedia Central
  • → Adobe AIR
  • → Vodophone Widgets
  • → Dashcode

Incompatible by design.

  • Apps that target a particular operating system won't run on another.
  • I'm a developer, and I don't care if its web or java or objective c.
  • I want to build apps that work offline.
  • That have access to device apis.
  • And I don't want to completely rewrite for new devices!

Standards?

  • System apps
  • DAP
  • Widget spec

Packaged web applications.

  • I'm excited about packaged web apps. Truly portable. Interop built in.
  • A community that understands the value of the web and demands standardization.
  • Lets talk about specific pkgd app characteristics to better understand...

Application manifest.

  • Sundry details: author, license, springboard title, version, description.
  • Config and prefs junk: webview bounce, status bar, spashscreens, and icons.
  • Origin (delivery, install, updates, uninstall).
  • Whitelisting the network.
  • Permissions for features.

Runtime context.

  • In the case of phonegap we run on the file:// protocol which x-domain issues.
  • Windowing, specifically window.open works differently.
  • System nofications vs network notifications.
  • Filesystem locations.
  • Background workers.

Excution lifecycle events.

  • Apps live a weird life compared to regular browser context.
  • Launch vs initialization.
  • Online, offline, and bandwidth changes.
  • Pause and resume.
  • System nofications such as battery events: tell the battery story.
  • Protected resource/service requests.

Runtime security beyond a sandbox.

  • Apps employ different trust models. Explicit vs reviewed.
  • Browsing context / navigation inside the runtime, in an InAppBrowser.
  • Network privacy.
  • API Security aka Permissions.
  • Data isolation and/or encrypted key store.
  • App interop by things like web intents or activties.

Device APIs today.

  • Camera, geo, accelerometer.
  • Filesystem, media, contacts.

Sure, this stuff is cool.

But we need to proceed with caution.
