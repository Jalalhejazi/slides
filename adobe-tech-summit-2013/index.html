    let me tell you some characteristics to better understand

    • Application manifest.

    • Sundry details: author, license, springboard title, version, description
    • Configuration and preferences junk: webview bounce, status bar, spashscreens, and icons
    • Origin (delivery, install, updates, uninstall)
    • Whitelist
    • Permissions

    Runtime context.

    In the case of phonegap we run on the file:// protocol which means cross domain not an issue.

  • windowing, specifically window.open works differently
  • nofications and alerts
  • filesystem locations

    • Excution lifecycle events.

    Apps live a weird life compared to regular browser context.

    • launch and initialization
    • online, offline, variable bandwidth changes
    • pause and resume
    • resume
    • protected resource/service requests
    • system nofications such as battery events: tell the battery story
    • remote nofications
    • background workers

    Runtime security beyond a sandbox.

    • Application Trust Model
    • Browsing Context Trust Model and Navigation inside the runtime, in an InAppBrowser
    • Network privacy
    • API Security aka Permissions
    • Data isolation or Private Storage
    • App interop by things like web intents or activties

    Device APIs today.

    • camera, geo, accelerometer
    • filesystem, media, contacts
    • websockets got nothing on raw sockets
